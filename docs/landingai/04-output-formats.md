# 04 — Output Formats

BCP produces **three deliverables** from a compliance session (plus dashboard data).

---

## Output 1 — Compliance report PDF

**Example:** `bcp-compliance2-report-2026-06-25.pdf`

**Contents:**

- Header: session name, date, documents compared
- Executive summary: overall % compliant, partial, non-compliant
- **Need-focus section:** only Partial + Non-Compliant points (plain lines)
- **Detail cards:** one per gov point
  - Point ID + title (e.g. **2.6.5** Corrective Training)
  - Badge: FULLY COMPLIANT / PARTIAL / NON-COMPLIANT
  - Confidence %
  - Evidence: internal file, page, section, quote
  - Fulfilled clauses (bullets)
  - Corrective action + responsibility (if gap)

**Generated by:** AI Lab batch → `/ai-lab-report` → Download PDF  
**Code:** `apps/web/src/lib/ai-lab/export-pdf.ts`

---

## Output 2 — Annotated internal PDF

**Purpose:** Original internal policy PDF with **visual mapping** — shows *where* each gov law point is satisfied.

**Contents:**

- Full unmodified internal document pages
- Per mapped gov point:
  - **Highlight rectangle** over evidence text (section / bullet)
  - **Callout box:** `2.6.5 | COMPLIANT | 100%` + full requirement text
  - Color: green (compliant), yellow (partial), red (non-compliant)

**Example mapping:**

| Gov point | Internal evidence location |
|-----------|----------------------------|
| 2.6.5 corrective training | Section 7.4 Training, bullet on immediate corrective action |

**Generated by:** Reference Mapper → Download annotated PDF  
**Code:** `apps/web/src/lib/ai-lab/export-reference-mapper-pdf.ts`

**Technical stack:**

- Text positions: `POST /ai/extractpdf-pages` (pdf-parse on API)
- Drawing: `pdf-lib` in browser

---

## Output 3 — Excel workbook

**Format:** Client sample template (`.xlsx`)

| Column | Field |
|--------|-------|
| Point ID | Gov requirement number |
| Requirement text | Full clause |
| Status | Compliant / Partial Compliant / Non-Compliant |
| Confidence % | 0–100 |
| Internal reference | File + section |
| Evidence quote | Verbatim from internal doc |
| Corrective action plan | If Partial/Non |
| Responsibility | Department |
| Target date | Remediation deadline |

**Generated by:** `/compliance` export or API `GET /excel/export/:sessionId`  
**Code:** `apps/api/src/modules/excel/`

---

## How the two PDFs relate

```
Same comparison session (compliance_items in DB)
        │
        ├─► Report PDF     = human-readable summary for auditors
        │
        └─► Annotated PDF  = proof inside the internal document itself
```

Both are required for your workflow:

1. **Report** — management / regulator pack (like attached sample)
2. **Annotated internal** — shows compliance *at the source* for policy owners

---

## Output file naming

| File | Pattern |
|------|---------|
| Report | `bcp-compliance-report-YYYY-MM-DD.pdf` |
| Annotated internal | `bcp-reference-mapper[-batch]-YYYY-MM-DD.pdf` |
| Excel | `bcp-compliance-YYYY-MM-DD.xlsx` |
